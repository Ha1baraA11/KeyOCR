## 最终方案详解

**算法：两遍贪心选择 + 尖峰过滤**

核心思路：
1. 计算相邻帧差异 diffs[i] = mean(|frame[i] - frame[i-1]|)
2. 候选帧条件：prev_diff 和 curr_diff 都较低（帧稳定），next_diff 较高（即将切换），ratio 在范围内
3. 孤立尖峰过滤：next_diff 远高于周围中位数的帧跳过（过滤场景切换等极端帧）
4. 贪心选择：按 score = ratio × next_diff 降序排列，满足 gap 约束就选中
5. 两遍执行：第一遍严格高置信，第二遍宽松补充

```python
# 第一遍: 严格条件 (高 ratio, 高 next_diff, 尖峰过滤)
# 候选条件: prev_diff <= 13, curr_diff <= 13, next_diff >= 12,
#           ratio in [1.5, 3.5], next_diff < context_median * 3.0
# 选择: 贪心按 score 降序, gap >= 18

# 第二遍: 宽松条件 (低 ratio, 低 next_diff, 无尖峰过滤)
# 候选条件: prev_diff <= 17, curr_diff <= 17, next_diff >= 11,
#           ratio in [1.2, 5.0]
# 选择: 贪心按 score 降序, gap >= 20, 与第一遍结果合并
```

**关键发现：**
- 单遍 spike 策略的瓶颈是"高分误选帧"（如场景切换产生的极端 next_diff）会阻塞正确帧
- 两遍策略让高置信帧先占位，再用宽松条件补充漏选
- 尖峰过滤（spike_mult=3.0）能有效去除孤立极端帧而不影响正确帧
- 贪心选择（按 score 排序）优于分组选择（按距离分组后选最高分）

    "frame_003138.png", "frame_003234.png", "frame_003354.png", "frame_003528.png",
    "frame_003594.png", "frame_003651.png", "frame_003768.png", "frame_003903.png",
    "frame_004017.png",
}

# 计算差异
diffs = [0]
prev_img = None
for i, fname in enumerate(files):
    img = cv2.imread(os.path.join(frames_dir, fname), cv2.IMREAD_GRAYSCALE)
    if img is None:
        continue
    if prev_img is not None:
        diff = np.mean(np.abs(prev_img.astype(float) - img.astype(float)))
        diffs.append(diff)
    prev_img = img
diffs = np.array(diffs)

def evaluate(name, selected_set):
    tp = len(selected_set & correct)
    fp = len(selected_set - correct)
    fn = len(correct - selected_set)
    p = tp / (tp + fp) if (tp + fp) > 0 else 0
    r = tp / (tp + fn) if (tp + fn) > 0 else 0
    f1 = 2 * p * r / (p + r) if (p + r) > 0 else 0
    print(f"{name}: 选={len(selected_set)} 命中={tp} 漏={fn} 多={fp} P={p:.1%} R={r:.1%} F1={f1:.1%}")
    return f1
